# NuClaw 教程渐进式写作规划

## 核心原则

1. **每章结尾提出问题** —— 让读者意识到当前实现的不足
2. **下一章解决问题** —— 明确展示基于前一版的修改
3. **源码对比** —— 用 diff 或表格展示代码变化
4. **递进关系清晰** —— 读者能直观看到演进过程

## 关键演进节点

### 从规则到 LLM
- **Step 3** → **Step 4**: 解决会话上下文问题
- **Step 4** → **Step 5**: 接入 LLM，语义理解替代规则匹配

### 从对话到行动
- **Step 5** → **Step 6**: LLM 能调用外部工具
- **Step 6** → **Step 8**: 工具异步执行 + 安全防护

### 从单 Agent 到多 Agent
- **Step 9** → **Step 11**: 工具注册表 → 多 Agent 协作

### 从 ephemeral 到 persistent
- **Step 4**: 会话级上下文（短时）
- **Step 16**: 跨会话长期记忆（持久）

### 从 Demo 到生产
- **Step 14**: 容器化
- **Step 15**: IM 接入
- **Step 17-20**: 完整 SaaS 平台
